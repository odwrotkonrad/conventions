# Claude Agents Convention

Every workspace repo carries two repo-local claude agents in `<repo>/.claude/agents/`: `RO-<Repo>` (read-only scoping, planning, review; model fable) and `RW-<Repo>` (executor; model sonnet). On virt machines only, che renders `<repo>/.claude/settings.json` making `RW-<Repo>` the repo's default main-thread agent. Each repo owns only its `che.yml` wiring; the shared snippets live in `configs`, fetched as remote renderTemplates sources. Rendered outputs never land in vcs.

## Naming

`RO-<Repo>` / `RW-<Repo>`, repo name PascalCase (`configs` → `Configs`, `infra/oci-images` → `OciImages`).

## Ownership Split

- `configs` owns the shared prose, one source per sentence, in `templates/agents/snippets/`: `ro.md`, `rw.md` (full agent files: frontmatter + body, ctx `.repo` PascalCase), `settings.json` (`{"agent": "RW-{{ .repo }}"}`), `pwd.md` (invocation constraint, included by both agent snippets), `claude-gitignore`.
- Each repo owns only its `che.yml` `claude-agents` profile: remote sources `@gitlab.com/konradodwrot/configs//templates/agents/snippets/<name>` with `ctx: {repo: <Repo>}`, the only per-repo content. No template files in the repo.

Remote sources resolve from the configs default branch at render time: no sibling checkout needed, snippet changes apply once pushed to configs main. The repo path is never hardcoded: snippets inject it at render time from the rendering cwd (`env.Getenv "PWD"`), che renders from the repo root.

## Frontmatter

`name`, `description`, `model`, `permissionMode: bypassPermissions`, `memory: local` (memory dir `.claude/agent-memory-local/<name>/`, stays out of vcs). `tools` omitted: inherit all.

## Wiring

Each repo's `che.yml` carries one `claude-agents` profile, `options: {autoExec: true, execIf: ['builtin:isVirt']}`, rendered by that repo's `make render-templates`: on virt it renders, elsewhere it is skipped. Agent and settings dests use `skipAutoGeneratedHeader: true` (the default overwrite prepends an autogen header, breaking frontmatter and JSON); the `.gitignore` dest keeps the default (the header is a valid gitignore comment). `settings.local.json` stays free for ad-hoc overrides.

## VCS Hygiene

Belt and suspenders: the global git ignore (`configs/root/HOME/.config/git/ignore`) ignores `**/.claude/settings.json`, `**/.claude/agents/{ro,rw}.md`, `**/.claude/agent-memory-local/` on all machines; the rendered per-repo `.claude/.gitignore` covers hosts without that global ignore.

## Example

`example/che.yml` is the consumer side: the `claude-agents` profile a repo carries, its entire footprint. The shared snippets live in `configs`, not here. On virt, `che render-templates` in the repo renders all four dests.
